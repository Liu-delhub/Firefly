# Firefly 博客全面傻瓜式说明书

这份说明书按“从第一个文件夹点到目标文件”的方式写。你以后想改博客时，先来这里找对应功能，再照着路径打开文件、复制代码、修改内容。

## 先记住项目结构

你的博客项目根目录叫：

```text
Firefly
```

以后说明书里的路径都从 `Firefly` 开始写，比如：

```text
Firefly > src > config > musicConfig.ts
```

意思是：

```text
先打开 Firefly 文件夹
再打开 src 文件夹
再打开 config 文件夹
最后打开 musicConfig.ts 文件
```

## 哪些文件夹能随便放资源

### `Firefly > public`

`public` 里的文件会原样出现在网站里。这里适合放音乐、视频、普通图片、favicon、相册图。

路径写法要注意：

真实文件位置：

```text
Firefly > public > assets > music > test.mp3
```

代码里写：

```ts
"/assets/music/test.mp3"
```

不要写：

```ts
"public/assets/music/test.mp3"
```

### `Firefly > src > assets`

`src/assets` 里的图片会被 Astro 处理和优化。默认壁纸、头像、Logo 多数放这里。

真实文件位置：

```text
Firefly > src > assets > images > DesktopWallpaper > d1.avif
```

代码里写：

```ts
"assets/images/DesktopWallpaper/d1.avif"
```

这里不要加开头的 `/`。

## 根目录每个文件夹是干嘛的

### `Firefly > .github`

GitHub 自动化配置。里面有自动检查、自动构建、自动部署相关文件。

平时不要改。

常见文件：

| 文件夹或文件 | 用途 |
|---|---|
| `.github > workflows` | GitHub Actions 自动化流程 |
| `.github > ISSUE_TEMPLATE` | GitHub issue 模板 |
| `.github > pull_request_template.md` | Pull Request 模板 |

### `Firefly > docs`

项目原作者自带的文档和图片。它不是网站正文内容，主要用于说明主题功能。

平时不用改。

### `Firefly > public`

公开静态资源目录。音乐、视频、相册、favicon、看板娘模型都在这里。

常见目录：

| 路径 | 用途 |
|---|---|

| `public > assets > music > auto > cover` | 自动匹配的音乐封面 |
| `public > assets > videos` | 背景视频 |
| `public > assets > images` | 公开图片 |
| `public > favicon` | 浏览器标签页图标 |
| `public > gallery` | 相册图片 |
| `public > pio` | 看板娘模型 |

### `Firefly > scripts`

脚本目录。这里的文件负责生成图标、生成图片占位图、创建新文章、构建字体。

一般不用改。

常见文件：

| 文件 | 用途 |
|---|---|
| `generate-icons.js` | 生成图标数据 |
| `generate-lqips.ts` | 生成图片低清占位图 |
| `new-post.js` | 新建文章脚本 |
| `new-dynamic.js` | 新建动态脚本 |
| `subset-fonts.ts` | 构建字体子集 |

### `Firefly > src`

网站源码目录。大部分博客功能都在这里。

这里最重要。

## 添加音乐

现在推荐用“自动识别文件夹”的方式。你以后只要把歌放进固定文件夹，播放器就会自动识别，不用再手写歌单代码。

### 第一步：确认使用本地音乐模式

先打开音乐配置文件：

```text
Firefly > src > config > musicConfig.ts
```

找到：

```ts
mode: "local",
```

保持这样，意思是使用本地音乐文件。

### 第二步：把歌曲放进自动识别文件夹

歌曲放这里：

```text
Firefly > public > assets > music > auto
```

比如你有一首歌：

```text
我的歌 - 歌手名字.mp3
```

就直接放成：

```text
Firefly > public > assets > music > auto > 我的歌 - 歌手名字.mp3
```

刷新开发服务或重新构建后，播放器就会自动识别成：

```text
歌名：我的歌
歌手：歌手名字
```

### 文件名格式

推荐这样命名：

```text
歌名 - 歌手.mp3
歌名 - 歌手.flac
```

例子：

```text
特别的人 - 方大同.flac
唯一 - G.E.M.邓紫棋.mp3
我和你 - 陈帝勋.flac
```

如果文件名没有 ` - `，系统也能识别，但歌手会显示成：

```text
未知歌手
```

支持的音乐格式：

```text
.mp3
.flac
.wav
.ogg
.m4a
.aac
```

### 添加封面

封面放这里：

```text
Firefly > public > assets > music > auto > cover
```

封面文件名要和歌曲文件名的主体一样。

比如歌曲是：

```text
Firefly > public > assets > music > auto > 我的歌 - 歌手名字.mp3
```

封面就放：

```text
Firefly > public > assets > music > auto > cover > 我的歌 - 歌手名字.webp
```

也可以用这些格式：

```text
.webp
.png
.jpg
.jpeg
.avif
```

### 添加歌词

歌词放这里：

```text
Firefly > public > assets > music > auto > lrc
```

歌词文件名也要和歌曲文件名的主体一样。

比如歌曲是：

```text
我的歌 - 歌手名字.mp3
```

歌词就放：

```text
Firefly > public > assets > music > auto > lrc > 我的歌 - 歌手名字.lrc
```

支持：

```text
.lrc
.txt
```

### 当前自动识别配置

打开：

```text
Firefly > src > config > musicConfig.ts
```

你会看到：

```ts
local: {
	autoScan: {
		enable: true,
		folder: "/assets/music/auto",
		coverFolder: "/assets/music/auto/cover",
		lrcFolder: "/assets/music/auto/lrc",
	},
	playlist: [],
},
```

意思是：

| 配置 | 作用 |
|---|---|
| `enable: true` | 开启自动扫描 |
| `folder` | 自动扫描歌曲文件夹 |
| `coverFolder` | 自动匹配封面文件夹 |
| `lrcFolder` | 自动匹配歌词文件夹 |
| `playlist: []` | 手动补充歌单，默认留空 |

### 删除音乐

进入：

```text
Firefly > public > assets > music > auto
```

删除对应歌曲文件即可。

如果有封面，也删除：

```text
Firefly > public > assets > music > auto > cover > 同名封面.webp
```

如果有歌词，也删除：

```text
Firefly > public > assets > music > auto > lrc > 同名歌词.lrc
```

### 手动补充特殊歌曲

一般不用手写。只有你想加网络歌曲、特殊封面、特殊歌词时，才写 `playlist`。

打开：

```text
Firefly > src > config > musicConfig.ts
```

找到：

```ts
local: {
	autoScan: {
		enable: true,
		folder: "/assets/music/auto",
		coverFolder: "/assets/music/auto/cover",
		lrcFolder: "/assets/music/auto/lrc",
	},
	playlist: [],
},
```

改成：

```ts
local: {
	autoScan: {
		enable: true,
		folder: "/assets/music/auto",
		coverFolder: "/assets/music/auto/cover",
		lrcFolder: "/assets/music/auto/lrc",
	},
	playlist: [
		{
			name: "网络歌曲名",
			artist: "歌手名字",
			url: "https://example.com/music.mp3",
			cover: "https://example.com/cover.webp",
			lrc: "",
		},
	],
},
```

这样最终歌单会包含：

```text
auto 文件夹自动识别的歌曲
playlist 里手动写的歌曲
```

## 使用网易云歌单

打开：

```text
Firefly > src > config > musicConfig.ts
```

找到：

```ts
mode: "local",
```

改成：

```ts
mode: "meting",
```

然后找：

```ts
meting: {
	api: "https://api.i-meto.com/meting/api?server=:server&type=:type&id=:id&r=:r",
	server: "netease",
	type: "playlist",
	id: "10046455237",
	auth: "",
}
```

只改 `id` 就行。

比如你的网易云歌单链接是：

```text
https://music.163.com/#/playlist?id=123456789
```

那么就写：

```ts
id: "123456789",
```

## 音乐常用设置

打开：

```text
Firefly > src > config > musicConfig.ts
```

### 默认音量

找到：

```ts
volume: 0.2,
```

意思是 20% 音量。

如果要 50%：

```ts
volume: 0.5,
```

### 自动播放

找到：

```ts
autoPlay: true,
```

打开网站自动播放。

关闭自动播放：

```ts
autoPlay: false,
```

浏览器可能会拦截自动播放声音。如果被拦截，需要用户点击页面后才会继续播放。

### 播放顺序

找到：

```ts
playMode: "list",
```

可选：

| 写法 | 作用 |
|---|---|
| `"list"` | 列表循环 |
| `"one"` | 单曲循环 |
| `"random"` | 随机播放 |

## 发布文章

文章已经支持自动识别。你把 `.md` 或 `.mdx` 文件放进文章目录，博客就会自动读取。

为了同一天多篇文章不乱，推荐统一放到自动文章文件夹：

```text
Firefly > src > content > posts > auto
```

里面再按日期建文件夹。

### 推荐文章文件夹格式

比如 2026 年 7 月 21 日写了两篇文章：

```text
Firefly > src > content > posts > auto > 2026-07-21 > 01-早上的记录.md
Firefly > src > content > posts > auto > 2026-07-21 > 02-晚上的记录.md
```

这样以后你自己找文章也很清楚。

### 第一步：新建日期文件夹

进入：

```text
Firefly > src > content > posts > auto
```

新建当天日期文件夹：

```text
2026-07-21
```

完整路径就是：

```text
Firefly > src > content > posts > auto > 2026-07-21
```

### 第二步：新建文章文件

在日期文件夹里新建：

```text
01-我的第二篇博客.md
```

如果同一天有第二篇：

```text
02-今天的第二篇.md
```

文件名前面的 `01`、`02` 是为了你自己看起来更清楚。真正排序主要看文章里的 `published` 时间。

### 第三步：复制文章模板

把下面内容复制进去：

```md
---
title: "我的第二篇博客"
published: 2026-07-21 10:30:00
description: "这是我的第二篇博客简介"
tags: ["日常"]
category: "日常"
draft: false
---

这里写正文。

这是第二段。
```

### 第四步：改文章信息

文章最上面这段叫 frontmatter：

```md
---
title: "我的第二篇博客"
published: 2026-07-21 10:30:00
description: "这是我的第二篇博客简介"
tags: ["日常"]
category: "日常"
draft: false
---
```

字段解释：

| 字段 | 怎么写 | 作用 |
|---|---|---|
| `title` | `"标题"` | 文章标题 |
| `published` | `2026-07-21 10:30:00` | 发布时间，建议写到具体时间 |
| `description` | `"简介"` | 首页文章卡片简介 |
| `tags` | `["日常", "记录"]` | 标签，可以多个 |
| `category` | `"日常"` | 分类，通常一个 |
| `draft` | `false` | 是否发布 |

### 同一天多篇文章怎么不乱

同一天多篇文章时，`published` 写到具体时间。

早上的文章：

```md
published: 2026-07-21 09:00:00
```

晚上的文章：

```md
published: 2026-07-21 21:30:00
```

博客会把时间更晚的文章排在更前面。如果两篇文章时间完全一样，就会按文件名排序，所以文件名建议用：

```text
01-标题.md
02-标题.md
03-标题.md
```

### 草稿和发布

发布：

```md
draft: false
```

草稿：

```md
draft: true
```

开发环境可能能看到草稿，正式部署一般不会显示草稿。

### 添加文章封面

推荐把文章图片放这里：

```text
Firefly > src > content > posts > images
```

比如放了：

```text
Firefly > src > content > posts > images > my-cover.avif
```

文章头部加：

```md
image: "images/my-cover.avif"
```

完整例子：

```md
---
title: "我的第二篇博客"
published: 2026-07-21 10:30:00
description: "这是我的第二篇博客简介"
image: "images/my-cover.avif"
tags: ["日常"]
category: "日常"
draft: false
---
```

### 在正文插入图片

如果图片放在：

```text
Firefly > public > assets > images > posts > test.png
```

文章正文里写：

```md
![图片说明](/assets/images/posts/test.png)
```

如果图片放在文章旁边或 `posts/images` 里，写法可能要根据文件位置调整。最简单的方法是把文章图片统一放到 `public > assets > images > posts`，正文路径就从 `/assets/images/posts/` 开始。

### 删除文章

进入：

```text
Firefly > src > content > posts > auto
```

找到对应 `.md` 文件，直接删除。

比如删：

```text
Firefly > src > content > posts > auto > 2026-07-21 > 01-我的第二篇博客.md
```

这篇文章就没了。

### 添加桌面壁纸



现在不需要再手动改配置。只要图片在这个文件夹里，网站就会自动扫描并加入手机端随机壁纸列表。

### 开关壁纸轮播

打开：

```text
Firefly > src > config > backgroundWallpaper.ts
```

找到：

```ts

	enable: true,
	interval: 5000,
	transitionEffect: "zoom",
}
```

关闭轮播：

```ts

```


过渡效果可选：

| 写法 | 效果 |
|---|---|
| `"fade"` | 淡入淡出 |
| `"zoom"` | 缩放 |
| `"slide"` | 滑动 |
| `"kenburns"` | 旋转木马 |

### 侧边栏位置

可选：

| 写法 | 效果 |
|---|---|
| `"left"` | 只显示左边 |

### 小组件顺序

在 `sidebarConfig.ts` 里找：

```ts
leftComponents: []
rightComponents: []
mobileBottomComponents: []
```

这里控制侧边栏有哪些小组件、顺序是什么。数组里谁写在前面，谁就显示在前面。

如果你删除某个名字，对应小组件就不会显示。

常见小组件：

| 名字 | 作用 |
|---|---|
| `profile` | 个人资料 |
| `announcement` | 公告 |
| `music` | 音乐 |
| `timeWeather` | 时间、访客地区、天气卡片 |
| `calendar` | 日历 |
| `categories` | 分类 |
| `tags` | 标签 |
| `stats` | 网站统计 |

### 时间天气卡

时间天气卡组件文件：

```text
Firefly > src > components > widget > TimeWeather.astro
```

右侧栏配置位置：

```text
Firefly > src > config > sidebarConfig.ts
```

现在它放在：

```text
最新动态 dynamic
时间天气 timeWeather
站点统计 stats
```

时间一定能显示。地区会尝试用 IP 粗略识别，天气会尝试用免 key 天气接口获取。如果接口失败，会显示“天气暂不可用”，不会影响网站打开。


## 修改相册

相册现在也支持自动识别。每一个子文件夹就是一本相册。

相册根目录：

```text
Firefly > public > gallery
```

### 新建一本相册

比如你要建一本 2026 年 7 月 21 日的旅行相册。

进入：

```text
Firefly > public > gallery
```

新建文件夹：

```text
2026-07-21-旅行
```

完整路径：

```text
Firefly > public > gallery > 2026-07-21-旅行
```

然后把图片放进去：

```text
Firefly > public > gallery > 2026-07-21-旅行 > 01.webp
Firefly > public > gallery > 2026-07-21-旅行 > 02.jpg
Firefly > public > gallery > 2026-07-21-旅行 > 03.png
```

博客会自动识别这本相册。

### 相册封面

如果你想指定封面，在相册文件夹里放一个叫 `cover` 的图片：

```text
Firefly > public > gallery > 2026-07-21-旅行 > cover.webp
```

支持：

```text
cover.webp
cover.jpg
cover.png
cover.avif
cover.gif
```

如果没有 `cover.*`，系统会自动用第一张图片当封面。

### 写相册名称、日期、地点、标签

在相册文件夹里新建：

```text
album.json
```

完整路径：

```text
Firefly > public > gallery > 2026-07-21-旅行 > album.json
```

内容这样写：

```json
{
  "name": "旅行相册",
  "description": "记录一次出门玩的照片",
  "date": "2026-07-21",
  "location": "重庆",
  "tags": ["旅行", "日常"]
}
```

字段解释：

| 字段 | 作用 |
|---|---|
| `name` | 相册显示名称 |
| `description` | 相册简介 |
| `date` | 相册日期，用于排序 |
| `location` | 地点 |
| `tags` | 标签，用于相册页筛选 |

如果不写 `album.json`，系统会用文件夹名当相册名。

### 删除相册

删除这本相册的整个文件夹即可：

```text
Firefly > public > gallery > 2026-07-21-旅行
```

### 手动覆盖相册信息

如果你想在配置里手动写相册信息，打开：

```text
Firefly > src > config > galleryConfig.ts
```

里面有：

```ts
albums: [
	{
		id: "firefly-2026",
		name: "可爱流萤",
		description: "飞萤之火自无梦的长夜亮起，绽放在终竟的明天。",
		location: "崩坏：星穹铁道",
		date: "2026-01-01",
		tags: ["崩坏星穹铁道", "流萤"],
	},
],
```

注意：

```text
id 必须和 public/gallery 里的文件夹名完全一样。
```

比如：

```text
Firefly > public > gallery > firefly-2026
```

那么配置里必须写：

```ts
id: "firefly-2026",
```

## 改完之后怎么确认没坏

只改文章、图片、音乐时，先运行：

```bash
pnpm check
```

准备部署时，运行：

```bash
pnpm build
```

如果 `pnpm check` 报错，通常是：

| 问题 | 常见原因 |
|---|---|
| 字符串没闭合 | 少了一个 `"` |
| 数组报错 | 少了英文逗号 `,` |
| 图标找不到 | 图标名写错 |
| 图片不显示 | 路径写错 |
| 文章不显示 | `draft: true` 或 frontmatter 写错 |

## 最容易写错的地方

### 英文逗号

数组里每一项后面都要有英文逗号：

```ts
[
	"第一项",
	"第二项",
]
```

对象之间也要有英文逗号：

```ts
{
	name: "第一首",
},
{
	name: "第二首",
},
```

### 路径开头

`public` 里的文件：

```ts
"/assets/music/test.mp3"
```

`src/assets` 里的文件：

```ts
"assets/images/DesktopWallpaper/d1.avif"
```

### 布尔值

开：

```ts
true
```

关：

```ts
false
```

不要写成：

```ts
"true"
"false"
```

## 不建议乱改的地方

| 路径 | 原因 |
|---|---|
| `Firefly > node_modules` | 依赖目录，不要手动改 |
| `Firefly > dist` | 构建输出，会被重新生成 |
| `Firefly > pnpm-lock.yaml` | 依赖锁定文件 |
| `Firefly > astro.config.mjs` | 项目核心配置 |
| `Firefly > tsconfig.json` | TypeScript 配置 |
| `Firefly > biome.json` | 格式化规则 |
| `Firefly > src > plugins` | Markdown 插件，逻辑复杂 |
| `Firefly > src > constants > icons.ts` | 自动生成 |
| `Firefly > src > constants > lqips.json` | 自动生成 |

## 给自己以后看的提醒

如果只是换文字、换链接、换图片、加文章，尽量别改 `components` 和 `utils`。

如果只是默认值不喜欢，优先改 `src/config`。

如果页面报错，先看报错里的文件路径和行号。大多数错误都是少逗号、引号不完整、路径写错、图标名不存在。

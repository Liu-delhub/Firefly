# Firefly 博客项目中文说明书

这份说明书是给以后自己修改博客用的。项目是一个基于 Astro、Svelte 和 TypeScript 的静态博客，部署时会把源码构建成静态网页。平时最常改的内容不是所有代码，而是配置文件、文章文件、图片音乐资源和少量布局组件。

## 先看这几个位置

| 你想改什么 | 优先打开哪里 |
|---|---|
| 网站标题、描述、主题色、页面开关 | `src/config/siteConfig.ts` |
| 主页壁纸、全屏壁纸、壁纸轮播、背景视频 | `src/config/backgroundWallpaper.ts` |
| 音乐播放器、歌单、本地音乐、自动播放、音量 | `src/config/musicConfig.ts` |
| 头像、昵称、个人简介 | `src/config/profileConfig.ts` |
| 导航栏菜单 | `src/config/navBarConfig.ts` |
| 侧边栏显示哪些小组件 | `src/config/sidebarConfig.ts` |
| 樱花、特效 | `src/config/effectsConfig.ts` |
| 发布文章、删除文章 | `src/content/posts` |
| 关于页、友链页、留言页正文 | `src/content/spec` |
| 放音乐、视频、普通图片 | `public/assets` |
| 放会被 Astro 优化的源码图片 | `src/assets` |

## 根目录文件夹

### `.github`

这里是 GitHub 自动化配置，比如自动检查代码、构建、部署。你平时不用改它。除非以后你要改 GitHub Actions 部署流程，否则不要动。

### `docs`

这里放项目自带文档和文档图片，不影响网站实际页面。你可以把它当作参考资料目录。

### `public`

这里放会原样复制到网站根目录的静态资源。路径写法通常不带 `public`。

比如文件真实位置是：

```text
public/assets/music/test.mp3
```

在配置里应该写：

```ts
"/assets/music/test.mp3"
```

常用子目录：

| 目录 | 作用 |
|---|---|
| `public/assets/music` | 本地音乐文件 |
| `public/assets/music/cover` | 音乐封面 |
| `public/assets/videos` | 主页背景视频 |
| `public/assets/images` | 公开图片，比如打赏图、特效图、广告图 |
| `public/favicon` | 浏览器标签页图标 |
| `public/gallery` | 相册图片 |
| `public/pio` | 看板娘模型资源 |

### `scripts`

这里是构建和创建内容用的脚本。

| 文件 | 作用 |
|---|---|
| `generate-icons.js` | 构建前生成图标常量 |
| `generate-lqips.ts` | 生成图片低清占位图 |
| `new-post.js` | 新建文章用的脚本 |
| `new-dynamic.js` | 新建动态用的脚本 |
| `subset-fonts.ts` | 构建后生成字体子集 |

平时如果只是写文章，不需要改这些脚本。

### `src`

这里是网站源码，是最重要的目录。页面、组件、配置、文章、样式都在这里。

## `src` 目录

### `src/assets`

这里放源码管理的图片，构建时可能会被 Astro 优化。壁纸默认放在这里：

| 目录 | 作用 |
|---|---|
| `src/assets/images/DesktopWallpaper` | 桌面端壁纸 |
| `src/assets/images/MobileWallpaper` | 手机端壁纸 |
| `src/assets/images/avatar.avif` | 头像 |
| `src/assets/images/firefly.png` | 导航栏 Logo |

如果你要改默认壁纸，可以把新壁纸放到对应目录，然后去 `src/config/backgroundWallpaper.ts` 修改路径。

### `src/components`

这里放网站组件。组件就是页面上的一个个小模块，比如导航栏、页脚、文章卡片、音乐播放器、搜索按钮。

常用子目录：

| 目录 | 作用 |
|---|---|
| `components/layout` | 页面布局组件，比如导航栏、页脚、文章卡片、侧边栏 |
| `components/widget` | 侧边栏小组件，比如个人资料、音乐、日历、标签 |
| `components/features` | 功能组件，比如音乐管理、樱花特效、背景视频、Live2D |
| `components/controls` | 页面控制按钮，比如主题切换、壁纸切换、返回顶部 |
| `components/comment` | 评论系统组件 |
| `components/pages` | 特定页面专用组件，比如番组、相册、动态 |
| `components/common` | 通用按钮、图片、分页、Markdown 渲染等 |

你平时不需要直接改很多组件。除非要改页面长相或交互行为，再来这里找。

### `src/config`

这里是最适合你以后修改的地方。这个项目很多功能都做成了配置项，改配置比改组件安全。

| 文件 | 主要作用 |
|---|---|
| `siteConfig.ts` | 网站标题、描述、主题色、页面开关、文章列表设置、Bangumi、Bilibili UID |
| `backgroundWallpaper.ts` | 壁纸模式、壁纸图片、背景视频、壁纸轮播、横幅文字 |
| `musicConfig.ts` | 音乐播放器、歌单来源、本地歌曲、默认音量、自动播放 |
| `effectsConfig.ts` | 樱花特效开关、数量、速度、透明度 |
| `profileConfig.ts` | 侧边栏个人资料、头像、名字、简介 |
| `navBarConfig.ts` | 顶部导航栏菜单 |
| `sidebarConfig.ts` | 左右侧边栏布局、显示哪些组件 |
| `friendsConfig.ts` | 友链数据 |
| `galleryConfig.ts` | 相册配置 |
| `commentConfig.ts` | 评论系统配置 |
| `footerConfig.ts` | 页脚自定义内容开关 |
| `FooterConfig.html` | 页脚自定义 HTML 内容 |
| `announcementConfig.ts` | 公告配置 |
| `analyticsConfig.ts` | 网站统计配置 |
| `coverImageConfig.ts` | 文章封面图配置 |
| `licenseConfig.ts` | 文章版权说明 |
| `sponsorConfig.ts` | 打赏页面配置 |
| `pioConfig.ts` | 看板娘配置 |
| `fontConfig.ts` | 字体配置 |
| `expressiveCodeConfig.ts` | 代码块样式配置 |
| `mermaidConfig.ts` | Mermaid 图表配置 |
| `plantumlConfig.ts` | PlantUML 图表配置 |
| `index.ts` | 把所有配置统一导出，通常不用改 |

### `src/constants`

这里放项目常量和自动生成文件。

| 文件 | 作用 |
|---|---|
| `constants.ts` | 全局常量，比如壁纸模式名、布局尺寸 |
| `icon.ts` | favicon 默认配置 |
| `icons.ts` | 构建生成的图标数据 |
| `lqips.json` | 构建生成的图片低清占位图数据 |

一般不要手动改 `icons.ts` 和 `lqips.json`，它们是脚本生成的。

### `src/content`

这里放真正的内容。

| 目录 | 作用 |
|---|---|
| `src/content/posts` | 博客文章 |
| `src/content/posts/images` | 文章配图 |
| `src/content/spec` | 特殊页面内容，比如关于、友链、留言 |
| `src/content/dynamic` | 动态页面内容 |

一个 `.md` 或 `.mdx` 文件通常就是一篇内容。删除文章就是删除对应 `.md` 文件。

### `src/i18n`

这里是多语言翻译。

| 文件或目录 | 作用 |
|---|---|
| `i18nKey.ts` | 所有翻译 key |
| `translation.ts` | 翻译读取函数 |
| `languages/zh_CN.ts` | 简体中文文本 |
| `languages/en.ts` | 英文文本 |
| `languages/ja.ts` | 日文文本 |

如果你看到页面上某段文字不是配置里来的，可能在这里。

### `src/layouts`

这里是大布局。

| 文件 | 作用 |
|---|---|
| `Layout.astro` | 最外层页面壳，负责 HTML 头部、全局样式、全局脚本、音乐和樱花 |
| `MainGridLayout.astro` | 主页面网格，负责导航栏、壁纸、侧边栏、主体内容、页脚 |

主页壁纸、侧边栏位置、页面整体结构，通常和 `MainGridLayout.astro` 有关。

### `src/pages`

这里是路由页面。Astro 会根据文件名生成网址。

| 文件或目录 | 对应页面 |
|---|---|
| `[...page].astro` | 首页和分页文章列表 |
| `posts/[...slug].astro` | 文章详情页 |
| `about.astro` | 关于页 |
| `archive.astro` | 归档页 |
| `tags/index.astro` | 标签页 |
| `categories/index.astro` | 分类页 |
| `friends.astro` | 友链页 |
| `guestbook.astro` | 留言板 |
| `bangumi.astro` | 番组计划 |
| `anime.astro` | 追番页 |
| `gallery/index.astro` | 相册首页 |
| `dynamic/index.astro` | 动态页面 |
| `rss.xml.ts` | RSS 输出 |
| `robots.txt.ts` | 搜索引擎规则 |

如果你想“关闭某个页面”，优先改 `src/config/siteConfig.ts` 里的 `pages` 配置，不要直接删页面文件。

### `src/plugins`

这里是 Markdown 和 HTML 处理插件。比如 Mermaid、PlantUML、GitHub 卡片、图片网格、阅读时间等。一般不要动，除非你要改 Markdown 渲染规则。

### `src/styles`

这里是全站样式。

| 文件 | 作用 |
|---|---|
| `main.css` | 全局主样式，很多基础效果都在这里 |
| `navbar.css` | 导航栏样式 |
| `markdown.css` | 文章正文样式 |
| `banner-title.css` | 首页横幅文字样式 |
| `waves.css` | 横幅底部水波纹 |
| `transition.css` | 页面切换动画 |
| `tags.css` | 标签页样式 |
| `categories.css` | 分类页样式 |
| `gallery.css` | 相册样式 |
| `anime-bangumi.css` | 番组和追番相关样式 |
| `widget-responsive.css` | 侧边栏小组件响应式样式 |

如果你只是改内容，不要碰样式。样式改错可能影响整个网站。

### `src/types`

这里是 TypeScript 类型定义。配置文件通常有对应的类型文件。比如：

```text
src/config/musicConfig.ts
src/types/musicConfig.ts
```

如果你给配置新增字段，最好也在对应 `types` 文件里加字段，否则类型检查可能报错。

### `src/utils`

这里是工具函数。

| 文件 | 作用 |
|---|---|
| `content-utils.ts` | 文章排序、筛选、上一篇下一篇、相关文章 |
| `date-utils.ts` | 日期格式 |
| `layout-utils.ts` | 判断首页、读取壁纸图片 |
| `setting-utils.ts` | 主题、壁纸、樱花、轮播等前端设置逻辑 |
| `image-utils.ts` | 图片路径、封面图、随机图处理 |
| `url-utils.ts` | URL 和路径处理 |
| `toc-utils.ts` | 文章目录生成 |
| `crypto-utils.ts` | 加密文章相关 |
| `responsive-utils.ts` | 响应式布局计算 |

你之前改壁纸模式、音乐自动播放、默认设置时，很多逻辑就在这里或和这里有关。

### `src/workers`

这里放 Web Worker。当前主要是樱花特效的计算逻辑，避免特效卡住页面。一般不用改。

## 常用修改方法

### 修改网站标题

打开 `src/config/siteConfig.ts`，找：

```ts
title: "一叶知秋枫",
subtitle: "Feng ye",
description: "这是我的博客噢喵~",
```

### 修改主题色

打开 `src/config/siteConfig.ts`，找：

```ts
themeColor: {
	hue: 275,
}
```

`hue` 是 0 到 360 的色相值。275 是偏紫色。

### 修改默认壁纸模式

打开 `src/config/backgroundWallpaper.ts`，找：

```ts
mode: "fullscreen",
```

常用值：

| 值 | 效果 |
|---|---|
| `"banner"` | 横幅壁纸 |
| `"fullscreen"` | 全屏壁纸 |
| `"overlay"` | 全屏透明 |
| `"none"` | 不显示壁纸 |

### 修改壁纸图片

打开 `src/config/backgroundWallpaper.ts`，找：

```ts
src: {
	desktop: [
		"assets/images/DesktopWallpaper/d1.avif",
	],
	mobile: [
		"assets/images/MobileWallpaper/m1.avif",
	],
}
```

桌面壁纸放到 `src/assets/images/DesktopWallpaper`，手机壁纸放到 `src/assets/images/MobileWallpaper`。

### 修改背景视频

视频文件放到 `public/assets/videos`，然后打开 `src/config/backgroundWallpaper.ts`，找：

```ts
playerUrl: [
	"/assets/videos/kuroha-lineart.mp4",
],
```

### 修改音乐

打开 `src/config/musicConfig.ts`。

如果要用本地音乐，把：

```ts
mode: "meting",
```

改成：

```ts
mode: "local",
```

音乐文件放到 `public/assets/music`，封面放到 `public/assets/music/cover`，然后改：

```ts
local: {
	playlist: [
		{
			name: "歌名",
			artist: "歌手",
			url: "/assets/music/歌曲.mp3",
			cover: "/assets/music/cover/封面.webp",
			lrc: "",
		},
	],
},
```

### 发布文章

在 `src/content/posts` 新建 `.md` 文件，比如：

```text
my-second-post.md
```

模板：

```md
---
title: "我的第二篇博客"
published: 2026-07-21
description: "这是简介"
tags: ["日常"]
category: "日常"
draft: false
---

这里写正文。
```

### 删除文章

去 `src/content/posts` 删除对应的 `.md` 文件。

### 修改关于页

打开：

```text
src/content/spec/about.md
```

### 修改友链页

友链页面内容在：

```text
src/content/spec/friends.mdx
```

友链数据通常在：

```text
src/config/friendsConfig.ts
```

### 关闭页面

打开 `src/config/siteConfig.ts`，找：

```ts
pages: {
	friends: true,
	sponsor: true,
	guestbook: true,
	bangumi: true,
	gallery: true,
	anime: true,
	dynamic: true,
},
```

把不想显示的页面改成 `false`。

## 命令说明

所有命令都用 pnpm。

| 命令 | 作用 |
|---|---|
| `pnpm dev` | 本地预览网站 |
| `pnpm check` | 检查 Astro 页面有没有错误 |
| `pnpm type-check` | 检查 TypeScript 类型 |
| `pnpm build` | 正式构建网站 |
| `pnpm preview` | 预览构建后的结果 |
| `pnpm new-post` | 创建新文章 |

改完代码后，建议至少运行：

```bash
pnpm check
```

如果准备部署，运行：

```bash
pnpm build
```

## 不建议乱改的位置

这些地方不是不能改，而是改错很容易报错：

| 位置 | 原因 |
|---|---|
| `node_modules` | 依赖目录，不要手动改 |
| `dist` | 构建输出目录，会被重新生成 |
| `src/plugins` | Markdown 插件逻辑复杂 |
| `src/constants/icons.ts` | 自动生成 |
| `src/constants/lqips.json` | 自动生成 |
| `pnpm-lock.yaml` | 依赖锁定文件 |
| `astro.config.mjs` | 项目核心配置 |
| `tsconfig.json` | TypeScript 路径和编译配置 |
| `biome.json` | 格式化和代码检查规则 |

## 修改前后的习惯

改之前，先确认你要改的是“内容、配置、样式、还是逻辑”。

| 类型 | 优先位置 |
|---|---|
| 内容 | `src/content` |
| 图片音乐视频 | `public/assets` 或 `src/assets` |
| 网站开关和默认值 | `src/config` |
| 页面长相 | `src/styles` 和 `src/components` |
| 页面逻辑 | `src/utils` 和 `src/components/features` |

如果你只是想换文字、换图片、加文章，尽量不要改 `src/components` 和 `src/utils`。如果你想改功能，再去动这些文件。
